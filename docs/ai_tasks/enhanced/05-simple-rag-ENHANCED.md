# Task 05 — Simple RAG [ENHANCED]

## Goal

Build a complete Retrieval-Augmented Generation (RAG) pipeline that combines semantic search with LLM generation, creating a system that answers questions grounded in your document collection.

---

After completing this task, you'll understand:

- **Why RAG reduces hallucination** — Grounding LLM in retrieved documents
- **Prompt construction patterns** — System prompts, context assembly, instruction clarity
- **Context windows and token limits** — Managing constraints in LLM context
- **End-to-end system integration** — Combining embedding, retrieval, and generation

## Requirements

**Input:**
- `question`: string (natural language question, e.g., "What is machine learning?")

**Process (5 stages):**
1. **Embedding** — Convert question to vector
2. **Retrieval** — Find similar documents (top-K)
3. **Context Assembly** — Format documents as readable context
4. **Prompt Construction** — Create prompt with context + question
5. **Generation** — Call LLM with full prompt, get answer

**Output:**
- `answer`: string (generated response grounded in documents)
- `sources`: array of (id, text, score) — Which documents were used
- `tokensUsed`: number — For cost tracking

---

## Why RAG Works

### The Problem: LLM Hallucination

```
Pure LLM (no RAG):
Q: "What does your system do?"
A: "I build AI robots and cure diseases"
   ↓ WRONG! The model made this up (hallucinated)

Risk: False information presented with confidence
```

### The Solution: Ground in Documents

```
RAG System:
Q: "What does your system do?"
   ↓ Search documents
Retrieved: ["Your system embeds text and searches vectors"]
   ↓ Create prompt with retrieved text
Prompt: "Based on: '...embeds text and searches vectors'
         Answer: What does your system do?"
   ↓ LLM respects context
A: "Your system embeds text and searches vectors"
   ✓ CORRECT! Answer is grounded in documents
```

**Result:** LLM respects the constraint of using only provided documents.

console.log(`\\n  🧩 Stage 3: Assembling context...`);\n    const context = assembleContext(sources);\n    console.log(`      Context size: ${context.length} chars`);\n    \n    // STAGE 4: Construct client.chat.completions.create({\n      model: 'gpt-3.5-turbo',\n      messages: [\n        {\n          role: 'system',\n          content: systemPrompt,\n        },\n        {\n          role: 0.95,      // Nucleus sampling\n    });\n    \n    const answer = response.choices[0].message.content || '';\n    const tokensUsed = response.usage?.total_tokens || 0;\n    \n    const duration =    answer,\n      sources: sources.map(s => ({\n        id: s.id,\n        text: s.text.substring(0, 150), // Truncate for readability\n        score: s.score,\n      })),\n      tokensUsed,\n      duration,\n    };\n  } catch (error) {\n    const duration = Date.now() - startTime;\n    \n    if (error instanceof Error) {\n      console.error(`\\n  ❌ RAG Pipeline Error: ${error.message}`);\n      throw new Error(`RAG failed: ${error.message}`);\n    }\n    \n    throw error;\n  }\n}\n```\n\n### Step 2: Create API endpoint\n\n```typescript\n// src/endpoints/api/rag.ts\nimport { Router, Flow\n\n```\n┌──────────────────────────────────────┐\n│ INPUT: Question (natural language)   │\n│ \"What is machine learning?\"          │\n└────────────┬─────────────────────────┘\n             │\n to vector using OpenAI embedding model\n- Same model used for document embeddings\n- Output: 1536-dimensional vector\n\n**Stage 2: Retrieval (50-100ms)**\n- Call Pinecone with question vector\n- Get top-K similar documents (K=3)\n- Each result includes: id, text, similarity score\n\n**Stage 3: Context Assembly (1ms)**\n- Format retrieved documents nicely\n- Preserve relevance ordering\n- Keep scores for reference\n\n**Stage 4: Prompt Construction (1ms)**\n- Create system prompt with rules\n- Insert retrieved documents\n- Add user question\n\n**Stage 5: Generation (1-2 seconds)**\n- Call GPT-3.5-turbo with full prompt\n- Model respects system prompt constraint\n- Returns generated answer\n\n---\n\n## Prompt Engineering Patterns\n\n### Pattern 1: Basic System
